# 坐月子小程序 — 产品需求文档（PRD）

**文档版本**：v1.0
**项目名称**：tinyDaysMinApp（沐沐记）
**技术基础**：微信小程序 + 微信云开发
**文档日期**：2026-06-17

---

## 1. 项目概述

### 1.1 背景

产后「坐月子」通常为 **28～42 天**，期间产妇需要饮食调理、休息恢复、情绪关注；新生儿需要喂养、睡眠、体温等日常照护。信息分散在微信群、笔记 App、纸质记录中，缺少一款**轻量、私密、可家庭协作**的专用工具。

### 1.2 产品定位

一款面向**产后妈妈及家人**的微信小程序，帮助用户：

- 记录并可视化「月子第几天」
- 管理每日饮食、护理、宝宝喂养等关键事项
- 获取分阶段、可操作的月子知识
- 与家人共享进度，减轻独自记事的负担

### 1.3 产品目标

| 目标 | 说明 |
|------|------|
| 降低记录成本 | 3 步内完成一次日常记录 |
| 提升安全感 | 分阶段提醒 + 权威知识，减少焦虑 |
| 促进家庭协作 | 伴侣/长辈可查看、协助记录 |
| 数据私密可控 | 本地 + 云端，用户可导出/删除 |

### 1.4 非目标（首版不做）

- 在线问诊、处方、电商交易
- 月嫂/月子中心撮合平台
- 复杂社区 UGC（评论、论坛）

---

## 2. 目标用户

| 用户角色 | 特征 | 核心诉求 |
|----------|------|----------|
| 新妈妈（主用户） | 产后 0～42 天，精力有限 | 快速记喂养/恢复，看今天该注意什么 |
| 伴侣 | 参与照护，非全程在场 | 了解进度，收到提醒，帮忙记录 |
| 长辈 | 关心饮食与禁忌 | 查看月子餐建议，避免「土法误区」 |
| 月嫂/亲友（可选） | 短期协助 | 只读或有限写入权限 |

---

## 3. 用户场景（User Stories）

1. **作为新妈妈**，我设置生产日期后，希望首页显示「月子第 N 天」和今日要点，以便知道当前阶段该注意什么。
2. **作为新妈妈**，我希望每次喂奶后 10 秒内记一次（时间、左右侧、时长），以便判断宝宝是否吃饱。
3. **作为家人**，我希望看到今日月子餐建议和禁忌，以便正确备餐。
4. **作为新妈妈**，我希望记录伤口/恶露/情绪，以便发现异常时就医有依据。
5. **作为伴侣**，我希望收到「该换尿布了」「该记录体温了」等提醒，以便分担照护。

## 4. 功能需求

### 4.1 模块总览

```
首页（月子日历）
├── 月子设置（预产期/生产日期、月子天数 28/30/42）
├── 今日概览（天数、待办、快捷记录入口）
├── 宝宝记录
│   ├── 喂养（母乳/配方/混合）
│   ├── 睡眠
│   ├── 排泄（大小便）
│   ├── 体温/黄疸（可选）
│   └── 生长（体重、身长）
├── 妈妈记录
│   ├── 身体恢复（恶露、伤口、疼痛评分）
│   ├── 情绪（简易量表）
│   ├── 用药/复查提醒
│   └── 饮水/休息打卡
├── 月子餐
│   ├── 分阶段食谱（第 1 周清淡 → 逐步进补）
│   ├── 今日推荐 + 禁忌清单
│   └── 收藏/自定义菜单
├── 知识库
│   ├── 分主题文章（哺乳、睡眠、情绪、禁忌）
│   └── 分天数推送（可选）
├── 提醒中心
│   ├── 订阅消息（喂养、用药、复查）
│   └── 家庭共享提醒
├── 数据统计
│   ├── 喂养/睡眠曲线（7 天/28 天）
│   └── 导出 PDF/图片（二期）
└── 我的
    ├── 宝宝档案 / 多胎切换
    ├── 家庭成员与权限
    ├── 隐私与数据管理
    └── 意见反馈
```

### 4.2 功能详述

#### F1 月子日历与设置（P0）

| 编号 | 需求 | 验收标准 |
|------|------|----------|
| F1.1 | 录入宝宝出生日期、可选预产期 | 支持修改，修改后重算月子天数 |
| F1.2 | 选择月子周期（28/30/42 天，可自定义） | 首页显示「第 N/总 M 天」及进度条 |
| F1.3 | 分阶段标签 | 自动划分：排恶露期 / 恢复期 / 进补期等（可配置文案） |
| F1.4 | 今日卡片 | 展示 3～5 条当日建议 + 未完成待办数 |

#### F2 宝宝日常记录（P0）

| 编号 | 需求 | 验收标准 |
|------|------|----------|
| F2.1 | 喂养记录 | 类型、开始时间、时长/奶量、左右侧（母乳） |
| F2.2 | 睡眠记录 | 开始/结束或一键「睡了/醒了」 |
| F2.3 | 排泄记录 | 大小便次数、性状（可选图标） |
| F2.4 | 快捷入口 | 首页 FAB 或底部 Tab，最近一次记录类型优先 |
| F2.5 | 时间线 | 按日聚合，支持编辑、删除（软删） |

#### F3 妈妈恢复记录（P0）

| 编号 | 需求 | 验收标准 |
|------|------|----------|
| F3.1 | 恶露 | 颜色/量（枚举 + 备注） |
| F3.2 | 伤口/疼痛 | 1～10 疼痛评分 + 文字备注 |
| F3.3 | 情绪 | 3～5 档表情或 PHQ-2 简化版 |
| F3.4 | 异常提示 | 持续高热、大量出血等关键词触发「建议就医」提示（非诊断） |

#### F4 月子餐（P1）

| 编号 | 需求 | 验收标准 |
|------|------|----------|
| F4.1 | 分阶段食谱库 | 至少 28 天 × 早中晚结构（可先做 7 天样板） |
| F4.2 | 禁忌清单 | 生冷、辛辣、酒精等分类展示 |
| F4.3 | 收藏与「今日吃了什么」 | 可勾选打卡 |

#### F5 知识库（P1）

| 编号 | 需求 | 验收标准 |
|------|------|----------|
| F5.1 | 分类浏览 | 哺乳、睡眠、情绪、饮食、宝宝护理 |
| F5.2 | 文章详情 | 标题、摘要、正文、来源标注 |
| F5.3 | 与天数联动 | 「月子第 N 天」推荐阅读（规则可配置） |

#### F6 提醒（P1）

| 编号 | 需求 | 验收标准 |
|------|------|----------|
| F6.1 | 喂养间隔提醒 | 用户设置间隔，订阅消息授权后推送 |
| F6.2 | 用药/复查 | 自定义时间，支持重复 |
| F6.3 | 静默时段 | 22:00～7:00 默认不推送（可改） |

#### F7 家庭共享（P2）

| 编号 | 需求 | 验收标准 |
|------|------|----------|
| F7.1 | 邀请家人 | 分享卡片 / 邀请码加入家庭组 |
| F7.2 | 权限 | 主账号：读写；成员：读 + 可选写 |
| F7.3 | 操作日志 | 谁记录了哪条（简要） |

#### F8 数据统计（P2）

| 编号 | 需求 | 验收标准 |
|------|------|----------|
| F8.1 | 喂养统计 | 日总次数、总时长/奶量 |
| F8.2 | 睡眠统计 | 日总睡眠时长 |
| F8.3 | 趋势图 | 近 7 天折线/柱状（echarts-for-weixin 等） |

#### F9 我的与系统（P0）

| 编号 | 需求 | 验收标准 |
|------|------|----------|
| F9.1 | 微信登录 | openid 静默登录 |
| F9.2 | 宝宝档案 | 昵称、性别、出生体重等 |
| F9.3 | 数据导出/删除 | 符合隐私合规，删除需二次确认 |
| F9.4 | 免责声明 | 内容仅供参考，不能替代医疗建议 |

## 6. 数据模型（概要）

基于云开发文档型数据库，建议集合：

| 集合 | 主要字段 |
|------|----------|
| `users` | openid, nickName, avatar, createTime |
| `families` | familyId, ownerOpenid, members[], babyIds[] |
| `babies` | babyId, name, birthDate, birthWeight, familyId |
| `confinement_settings` | userId/familyId, startDate, totalDays, deliveryType |
| `baby_records` | type(feed/sleep/diaper/...), babyId, time, payload, creator |
| `mom_records` | type(lochia/pain/mood/...), userId, date, payload |
| `meal_plans` | dayIndex, meals[], tags, content（静态可放云存储 JSON） |
| `articles` | category, title, content, dayRange |
| `reminders` | userId, type, cron/interval, enabled |

## 7. 非功能需求

| 类别 | 要求 |
|------|------|
| 性能 | 首屏 < 2s（4G）；列表分页，单次 ≤ 20 条 |
| 可用性 | 核心记录流程 ≤ 3 次点击；大按钮、高对比 |
| 兼容性 | 微信基础库 2.20+；iOS / Android 主流机型 |
| 安全 | 云函数鉴权；家庭数据按 familyId 隔离 |
| 隐私 | 《隐私政策》《用户协议》；最小化采集；敏感健康数据加密存储（二期） |
| 合规 | 健康内容标注来源；不做医疗诊断表述 |
| 离线 | 记录可先写本地缓存，联网后同步（P2） |

## 8. 技术方案建议

与现有仓库对齐：

| 层级 | 方案 |
|------|------|
| 前端 | 微信小程序原生（当前项目为原生） |
| 后端 | 微信云开发：云函数 + 云数据库 + 云存储 |
| 鉴权 | 云函数内校验 openid；家庭组 membership 校验 |
| 推送 | 订阅消息模板（喂养提醒、复查提醒） |
| 内容 | 月子餐/文章：首期 CMS 手工录入或 JSON 静态包 |
| 图表 | echarts-for-weixin（统计模块） |

**与现有代码关系**：需替换 QuickStart 首页，新增业务 pages，扩展 `cloudfunctions` 业务云函数（可基于 `quickstartFunctions` 重构）。

## 11. 风险与依赖

| 风险 | 应对 |
|------|------|
| 健康内容合规 | 内容医学顾问审核；明确免责声明 |
| 用户精力不足 | 极简交互、默认值、语音输入（远期） |
| 家庭协作隐私 | 明确权限与退出机制 |
| 订阅消息触达率 | 关键节点引导授权，避免滥发 |

**外部依赖**：微信小程序账号、云开发环境、订阅消息模板审核、隐私政策页面托管。

## 12. 待确认事项（需产品决策）

1. **月子天数默认**：28 天还是 42 天？是否区分顺产/剖腹产不同建议？
2. **主用户画像**：只服务妈妈，还是宝宝记录为主、妈妈为辅？
3. **内容来源**：自研文案、合作机构、还是转载需授权素材？
4. **是否商业化**：纯工具免费，还是后期会员（高级统计/专属食谱）？
5. **品牌名**：是否沿用 `tinyDaysMinApp` /「沐沐记」等命名？
